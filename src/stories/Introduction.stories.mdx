# loyalreactdatePicker 0.7.5

<div className="subheading">Features</div>
<ul>
	<li>You can customize everything in this datepicker different ways.</li>
	<li>Pass customized dates</li>
	<li>Different modes: partial, single, interval, week</li>
	<li>Flexible configuration</li>
	<li>Much more amazing and real cool things</li>
	<li>Multi-languages support, just pass locale string</li>
</ul>

<div className="subheading">Requirements</div>

- classnames >= 2.3.1
- React >= 18.0.0


<div className="subheading">Installation</div>

```
npm i loyalreactdatepicker

import { DatePicker } from "loyalreactdatepicker";

import "loyalreactdatepicker/dist/datePicker.css";
```

<a href="#changelog">
	<h2 id="changelog">ChangeLog</h2>
</a>

<div className="subheading">0.7.5</div>

- fix up some issues with maxDate;
- fix up stories with max date;
- tech dept (remove extra dev dependencies, move some logic to hooks, clean up code)

<div className="subheading">0.7.4</div>

- interval mode changes: if you select dates after the last selected date or the first one, the interval will not be reset. It will be dropped only if you select selected date
- fix count of days available in month view. Now its always 42.
- remove useless useEffect
- fix bug with toggle december-january, january-december on month view. Year update didnt work correct.
- add min-height for datePicker wrapper (380px).

<div className="subheading">0.7.2</div>

- change width of datepicker header;
- fix issue with incorrect unselect date chosen by default on partial days select;
- fix issue with incorrect decade toggle and year pagination.
- add maxDate prop

<div className="subheading">0.7.1</div>

- move css to separate files.
- move style initialization to modules from components.
- fix bug with partial days select.
- change default onChange for week mode select. Before it returned one date, now it returns object `{start: Date (monday), end: Date(sunday)}`;
- move up state with selectedDates from MonthView to DatePicker root component.
- fix some bugs with decade/year movement (year or decade didnt saved).

<div className="subheading">0.7.0</div>

- fix year toggle;
- Remove selectedDates, selectedInterval and date prop. they were replaced to value prop. It can be Array of Dates or Date or `{start: Date | null, end: Date | null}`
- Move month date handlers to MonthViewComponent
- Change storybook examples realization.

<div className="subheading">0.6.2</div>

- WeekendDates renamed to WeekendDays. Also was changed a type of it. before: Array day numbers. Now: object: `{weekendDays: Array<daynumbers>, shouldBeDisabled: boolean}`
- Add className to customize weekendDays

<div className="subheading">0.6.1</div>

- Remove max-width for month body

<div className="subheading">0.6.0</div>

- documentation fixes;
- add today className for month view
- add footer element customization
- add isVisible boolean to hide/show calendar when you want

<div className="subheading">0.5.0</div>

- stable pre-release version (change rollup config & resolve some critical bugs)

<div className="subheading">0.3.0</div>

- add type "button" for period toggle button (month/year/decade label). It is important for datepicker in forms usage.

<div className="subheading">0.2.0</div>

- Decrease bundle size from 160kb to 140kb

<div className="subheading">0.1.0</div>

- pre-release
